## `evaluation_analysis` 最终精简版保留指标表

当前状态说明：

- 本表已按当前代码实现同步更新。
- 已落地的指标保留在表中。
- 容易与现有统计重复的别名指标已删除，不再保留双份命名。
- `evaluation_analysis` 只负责公式、规则、派生诊断和决策 flags；原始几何事实、语义栅格事实、ROI 候选生成不在本模块内生产。

| 一级类别 | 指标名                    | 类型  | 公式说明                                                     | 说明                     |
| -------- | ------------------------- | ----- | ------------------------------------------------------------ | ------------------------ |
| 总体结论 | `overall_score`           | float | benchmark 模式下由 `0.40·AP50 + 0.35·AP75 + 0.25·F1@0.50` 归一化得到；reference/online 模式下由参考误差分与在线质量分混合得到 | 综合质量总分，核心总指标 |
| 总体结论 | `quality_pass_flag`       | bool  | `quality_pass_flag = (overall_score ≥ T_pass)`               | 是否达到可接受质量阈值   |
| 总体结论 | `need_local_refine_flag`  | bool  | 当前由 `continue_refinement`、`candidate_rois` 或 `candidate_roi_count` 推导 | 是否需要局部细化         |
| 总体结论 | `need_param_search_flag`  | bool  | `need_param_search_flag = (overall_score < T_param) AND (quality_pass_flag = false)` | 是否需要重新做参数搜索   |
| 总体结论 | `need_finetune_flag`      | bool  | `need_finetune_flag = (overall_score < T_finetune) AND (quality_pass_flag = false) AND (regression_flag = false)` | 是否建议进入微调         |
| 总体结论 | `need_manual_review_flag` | bool  | `need_manual_review_flag = (failure_confidence < T_conf) OR (semantic_instance_conflict_flag = true)` | 是否建议人工复核         |

### A. 仅依赖模型输出几何/掩码本身的核心指标

| 一级类别   | 指标名                    | 类型  | 公式说明                                                     | 说明                       |
| ---------- | ------------------------- | ----- | ------------------------------------------------------------ | -------------------------- |
| 输出规模   | `pred_instance_count`     | int   | `pred_instance_count = N_inst`                               | 最终实例总数               |
| 输出规模   | `empty_output_flag`       | bool  | `empty_output_flag = (N_inst = 0) OR (pred_cover_ratio < T_empty_cover)` | 是否几乎无有效输出         |
| 覆盖质量   | `pred_cover_ratio`        | float | `pred_cover_ratio = Area(⋃ masks_pred) / Area(valid_DOM_region)` | 预测树冠总覆盖率           |
| 几何有效性 | `valid_instance_ratio`    | float | `valid_instance_ratio = N_valid_geom / N_raw_features`       | 几何有效实例比例           |
| 形状异常   | `shape_anomaly_ratio`     | float | 由极小冠幅比例、极小面积比例和单块主导占比组合得到           | 形状异常比例               |
| 小碎片     | `small_fragment_ratio`    | float | 当前统一映射自 `<4m²` 小碎片比例                            | 过小碎片实例比例           |
| 大斑块     | `large_blob_ratio`        | float | 当前由 `large_width_ratio_gt_6m` 与 `max_instance_area_share` 组合近似 | 异常大斑块比例，常反映粘连 |
| 重复重叠   | `duplicate_overlap_ratio` | float | 当前由 `overlap_pair_count / pred_instance_count` 近似       | 重复预测比例               |
| 边缘伪影   | `edge_artifact_score`     | float | 当前直接映射/归一化自 `edge_touch_ratio`                    | 边缘伪影程度               |
| 碎片化     | `fragmentation_score`     | float | `fragmentation_score = α·small_fragment_ratio + β·shape_anomaly_ratio` | 碎片化综合分数             |
| 粘连       | `merge_blob_score`        | float | `merge_blob_score = γ·large_blob_ratio + δ·(1 - semantic_instance_consistency)` | 粘连综合分数               |

### B. 依赖语义图与实例结果的核心指标

| 一级类别   | 指标名                            | 类型  | 公式说明                                                     | 说明                                   |
| ---------- | --------------------------------- | ----- | ------------------------------------------------------------ | -------------------------------------- |
| 语义一致性 | `semantic_instance_consistency`   | float | 当前统一使用语义 mask 与实例 union 的 `overlap_iou`         | 语义图与实例并集结果的一致性，核心指标 |
| 语义一致性 | `semantic_coverage_gap`           | float | 当前统一使用 `semantic_gap` 比例                             | 语义 mask 未被实例覆盖的缺口比例       |
| 语义一致性 | `semantic_instance_conflict_flag` | bool  | `semantic_instance_conflict_flag = (semantic_instance_consistency < T_sem_iou) OR (semantic_coverage_gap > T_sem_gap)` | 是否存在明显语义-实例冲突              |

### C. 依赖 COCO / 人工标注真值的核心指标

| 一级类别 | 指标名                 | 类型  | 公式说明                                                     | 说明                   |
| -------- | ---------------------- | ----- | ------------------------------------------------------------ | ---------------------- |
| 标注评估 | `AP50`                 | float | 当前代码字段名为 `ap50`                                      | 最核心标准分割指标之一 |
| 标注评估 | `AP75`                 | float | 当前代码字段名为 `ap75`                                      | 更严格的分割质量指标   |
| 标注评估 | `f1_score50`           | float | `f1_score50 = 2·precision50·recall50 / (precision50 + recall50)` | 兼顾精确率与召回率     |
| 标注评估 | `mean_iou_matched`     | float | `mean_iou_matched = (1/M)·Σ IoU(pred_i, gt_match_i)`         | 匹配实例平均 IoU       |

### D. 错误分解核心指标

| 一级类别 | 指标名                     | 类型  | 公式说明                                                     | 说明                     |
| -------- | -------------------------- | ----- | ------------------------------------------------------------ | ------------------------ |
| 错误分解 | `under_segmentation_score` | float | 当前按“一个 pred 同时覆盖多个 GT”的比例近似                  | 欠分割程度，反映树冠粘连 |
| 错误分解 | `over_segmentation_score`  | float | 当前按“一个 GT 被多个 pred 覆盖”的比例近似                   | 过分割程度               |
| 错误分解 | `miss_detection_score`     | float | 有真值时：`miss_detection_score = false_negative_count / N_gt`；无真值时可退化为语义支持缺失比例 | 漏检程度                 |
| 错误分解 | `false_detection_score`    | float | 有真值时：`false_detection_score = false_positive_count / N_pred`；无真值时可退化为语义越界比例 | 误检程度                 |
| 错误分解 | `failure_confidence`       | float | `failure_confidence = max(under_segmentation_score, over_segmentation_score, miss_detection_score, false_detection_score)` 归一化后与次大值拉开差距的函数 | 当前错误判断的置信度     |

### E. 坏区域 / 局部细化定位指标

| 一级类别 | 指标名                         | 类型  | 公式说明                                                     | 说明                     |
| -------- | ------------------------------ | ----- | ------------------------------------------------------------ | ------------------------ |
| 错误定位 | `local_refine_candidate_count` | int   | 当前统一映射自 `candidate_roi_count`                         | 局部细化候选区域数量     |

### F. 主模型 / 专家模型 / 优化前后对比指标

| 一级类别 | 指标名                      | 类型 | 公式说明                                                     | 说明             |
| -------- | --------------------------- | ---- | ------------------------------------------------------------ | ---------------- |
| 结果对比 | `accepted_improvement_flag` | bool | 当前用于专家模型结果接收和微调前后对比                       | 是否接受新结果   |
| 结果对比 | `regression_flag`           | bool | 当前用于专家模型结果接收和微调前后对比                       | 是否出现明显退化 |

### 当前不再保留的重复指标

- `false_negative_count`
  - 与 `fn50`/`miss_detection_score` 信息高度重合，当前不再单独保留为核心指标。
- `bad_block_count`
- `bad_region_area_ratio`
- `high_risk_block_exists_flag`
  - 这三项要求 `evaluation_analysis` 自己维护 block 级定位结果，和当前“ROI 候选生成归 data_processing，评估层只消费候选事实”的边界不一致，现阶段不保留。
