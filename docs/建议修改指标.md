按**“建议修改，但不一定全是致命错误”**来列。判断原则是：凡是会影响科学解释、跨样地可比性、流程决策稳定性、论文表述严谨性的指标，都建议修改。

核心依据有三点：

1. **标准评价指标**应尽量采用 IoU、Precision、Recall、F1、AP50/AP75、MAE、RMSE、R² 等通用指标；IoU、Precision、Recall、F1 在树冠分割研究中也被直接使用。
2. **单木树冠分割领域目前仍缺少完全统一的评价标准**，因此项目内部的权重、阈值、综合分可以保留，但必须写成“工程启发式/流程决策指标”。ISPRS 2024 ITC contest 也明确提到该领域存在评价标准不足的问题。
3. **郁闭度/冠层覆盖率类指标必须统一到“树冠垂直投影面积占地表比例”这个口径**，不能把不同量纲的“郁闭度、覆盖度、百分数、0–1 比例”混用。

## 一、在线输出质量类指标

| 建议修改指标                                                 | 现有问题                                                     | 科学、可行的修改建议                                         |
| ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `quality_score / 在线输出质量总分`                           | 当前公式把 `coverage_ratio`、`instance_leakage`、`semantic_gap`、`small_fragment_ratio`、`large_blob_ratio`、`1-instance_height_support_ratio` 直接加权，但各项可能不是统一 0–1 方向；无 CHM 时高度项会误伤 DOM-only 结果。 | 改成统一的风险分：`online_risk_score = Σ w_i * valid_norm_risk_i`，所有分项先 `clamp01()`。无 CHM 时移除高度项并重归一化权重。最终质量分再写成：`online_quality = 1 - online_risk_score`。 |
| `coverage_ratio = instance_union_area / semantic_area`       | 当 `semantic_area=0` 时除零；当语义为空但实例不为空时，该比值没有正常解释。 | 改成条件公式：`semantic_area>eps` 时计算；否则输出 `NA`，并新增两个 flag：`semantic_empty_flag`、`instance_present_without_semantic_flag`。 |
| `semantic_recall = overlap_area / semantic_area`             | 当语义面积为 0 时无定义。                                    | 保留，但只在 `semantic_area>eps` 时计算；否则设为 `NA`，不进入综合分。 |
| `instance_leakage = max(instance_union_area-overlap_area,0)/instance_union_area` | 当 `instance_union_area=0` 时除零。                          | 改成：`instance_union_area>eps` 时计算；否则设为 `NA` 或 0，同时由 `empty_output_flag` 单独处理空输出。 |
| `semantic_gap = max(semantic_area-overlap_area,0)/semantic_area` | 同样存在 `semantic_area=0` 问题。                            | 改成条件计算；语义为空时不参与质量分。                       |
| `overlap_iou`                                                | IoU 公式本身标准，但 union 为 0 时会除零。                   | 保留公式：`IoU = overlap_area / union_area`；当 `union_area<=eps` 时设为 `NA`。IoU 是分割精度评价的标准核心公式。([施普林格自然链接](https://link.springer.com/article/10.1007/s11676-025-01929-5)) |
| `semantic_instance_consistency = overlap_iou`                | 直接等于 IoU 可以，但命名容易让人误以为是模型语义置信度。    | 建议改名为 `semantic_instance_iou`，更直接。若要保留 consistency，应写成：`semantic_instance_consistency = clamp01(overlap_iou)`。 |

## 二、几何合理性与碎片/粘连类指标

| 建议修改指标                                                 | 现有问题                                                     | 科学、可行的修改建议                                         |
| ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `small_fragment_ratio_lt_4m2`                                | 固定 4 m² 阈值不适合所有树种、龄组、分辨率。幼林、密植林、经济林可能会把真实小冠幅误判为碎片。 | 改成相对阈值：`area_i < β * expected_crown_area`，例如 `β=0.25`；无小班参考冠幅时，用训练集/验证集树冠面积 P5 或 P10 作为阈值。 |
| `tiny_width_ratio_lt_1m / lt_2m`                             | 固定 1m、2m 阈值同样不通用。                                 | 改成：`eq_width_i < max(min_width_by_resolution, β * expected_mean_crown_width)`。例如 `β=0.4`。 |
| `large_width_ratio_gt_6m`                                    | 固定 6m 会误判。对成熟阔叶林 6m 可能正常，对幼林则可能是粘连。 | 改成：`width_outlier_ratio = mean(eq_width_i > α * expected_mean_crown_width)`，建议 `α=1.8~2.5` 作为初值，再用验证集校准。 |
| `max_instance_area_share = max(area_i)/union_area_m2`        | 公式可用，但无实例时会出错；且单独使用容易把真实大树冠误判为粘连。 | 加条件：无实例时设为 `NA`；并与 `width_outlier_ratio`、`semantic_instance_iou` 联合判断粘连。 |
| `top5_instance_area_share`                                   | 可解释性强，但不宜进入综合分，否则大冠层林分会被误罚。       | 建议只作为报告解释指标，不直接进入 online quality。          |
| `duplicate_overlap_ratio = overlap_pair_count / pred_instance_count` | 分母逻辑不合理。重叠对数最大为 `N*(N-1)/2`，除以 `N` 可能大于 1。 | 改成二选一：`duplicate_pair_ratio = overlap_pair_count / max(N*(N-1)/2,1)`；或 `overlapped_instance_ratio = overlapped_instance_count / max(N,1)`。 |
| `shape_anomaly_ratio`                                        | 由多个经验项加权，但 `dominant_blob_norm` 未明确定义。       | 必须补充定义。建议：`dominant_blob_norm = clamp01(max_instance_area_share / dominant_share_thr)`，其中 `dominant_share_thr` 可设为验证集 P95。 |
| `fragmentation_score`                                        | `0.65*small_fragment + 0.35*shape_anomaly` 是启发式，不是标准公式。 | 可保留，但改名为 `fragmentation_risk_score`；权重写成默认值，并允许通过验证集调参。 |
| `large_blob_ratio`                                           | 当前用 `max(large_width_ratio_gt_6m, max_instance_area_share)`，把冠幅异常和面积占比直接混合，解释性不够严谨。 | 改成：`large_blob_risk = max(width_outlier_ratio, dominant_area_risk)`；其中 `dominant_area_risk = clamp01(max_instance_area_share / P95_share)`。 |
| `merge_blob_score`                                           | 当前依赖 `1-semantic_instance_consistency`，但如果语义和实例都偏差，可能误判；且缺失时无法计算。 | 改成：`merge_blob_risk = weighted_mean(valid_terms=[large_blob_risk, duplicate_pair_ratio, 1-semantic_instance_iou])`。缺失项剔除后重归一化。 |

树冠分割研究中，过分割、欠分割、误检、漏检通常需要结合 IoU、false positive、false negative 来解释，而不是只靠一个固定面积阈值。

## 三、高度一致性类指标

| 建议修改指标                            | 现有问题                                                     | 科学、可行的修改建议                                         |
| --------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `instance_height_support_ratio`         | 当前 support 定义为 CHM>1m。这个阈值对成熟乔木合理，但对幼林、灌木、低矮经济林不一定合理。 | 改成可配置阈值：`height_support_thr_m`。默认可设 1m，但在报告中说明仅用于过滤低矮非树冠支撑。若有树种/龄组信息，应按林分类型设置。 |
| `height_mean / height_p95 / height_std` | 作为描述指标合理，但不应直接当作“质量好坏”。                 | 建议仅作为解释指标；如果用于质量判断，应转成与参考树高或 CHM 局部分布相关的相对异常值。 |
| `height_edge_strength`                  | 科学上可解释，但没有标准阈值。                               | 建议作为专家模型/后验解释指标，不进入主评分；或通过验证集学习阈值。 |

CHM/DSM 与 RGB 融合确实可提高树冠和树高提取表现，但不同研究中的阈值依赖数据、树种和高度范围，不能固定成通用常数。

## 五、Benchmark 标准真值指标

| 建议修改指标              | 现有问题                                                     | 科学、可行的修改建议                                         |
| ------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `AP50 / AP75`             | 公式方向正确，但建议不要只报 AP50/AP75。                     | 正式 benchmark 建议同时报告：`AP@[0.50:0.95]`、`AP50`、`AP75`、`Precision`、`Recall`、`F1@0.50`、`mean matched IoU`。COCO 数据集本身包含 instance segmentation mask，并用于检测/分割评估。 |
| `Precision / Recall`      | 当前公式没问题，但需要明确是“实例级”还是“面积级”。           | 对实例分割 benchmark，建议使用实例级：`precision = TP/(TP+FP)`，`recall = TP/(TP+FN)`；面积级 Precision/Recall 可以作为补充。 |
| `f1_score50`              | 公式标准，但当 `precision+recall=0` 时会除零。               | 改成：若 `P+R>0`，`F1=2PR/(P+R)`；否则 `F1=0`。F1 是树冠分割研究中常见的综合指标。 |
| `mean_iou_matched`        | 只对 TP 计算会导致“漏检严重但匹配质量高”的情况被掩盖。       | 保留，但不能单独作为总体质量指标；必须与 Recall/FN 一起解释。 |
| `MAE / RMSE / RMSE% / R²` | 公式合理，但应说明只对 matched crowns 计算，不能代表所有 GT。 | 保留，同时输出 `num_matched_crowns`。如果匹配数量太少，应标记 `area_regression_unreliable_flag`。 |

## 六、错误分解类指标

| 建议修改指标                                            | 现有问题                                                     | 科学、可行的修改建议                                         |
| ------------------------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `under_segmentation_score = pred_multi_gt / pred_count` | “一个预测对应多个 GT”的判定规则不明确；`pred_count=0` 时除零。 | 定义规则：若一个预测实例与两个及以上 GT 的 `intersection_over_gt_area >= τ`，则计为 under-seg。建议 `τ=0.1~0.3`，通过验证集校准。 |
| `over_segmentation_score = gt_multi_pred / gt_count`    | “一个 GT 被多个预测覆盖”的判定规则不明确；`gt_count=0` 时除零。 | 定义规则：若一个 GT 被两个及以上预测实例覆盖，且每个预测与该 GT 的覆盖比例超过 `τ`，则计为 over-seg。 |
| `miss_detection_score = FN50 / GT_count`                | `GT_count=0` 时除零。                                        | `GT_count>0` 时计算；否则设为 0 或 `NA`。                    |
| `false_detection_score = FP50 / pred_count`             | `pred_count=0` 时除零。                                      | `pred_count>0` 时计算；否则设为 0 或 `NA`。                  |
| `failure_confidence`                                    | 当前公式 `top + 0.35*(top-second)` 把“错误严重度”和“错误类型确定性”混在一起。 | 拆成两个指标：`failure_severity = top_error_score`；`failure_type_confidence = clamp01(top_error_score - second_error_score)`。这样更适合决策。 |

## 七、ROI 细化决策类指标

| 建议修改指标                                   | 现有问题                                                     | 科学、可行的修改建议                                         |
| ---------------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `current_score / previous_score / improvement` | 如果 score 是 error score，则 `previous-current` 正值代表提升；如果是 quality score，则方向相反。 | 明确命名：`current_error_score`、`previous_error_score`、`error_reduction = previous_error_score - current_error_score`。 |
| `metric_thresholds`                            | 表中数量误差、冠幅误差、郁闭度误差阈值是经验值，不是文献标准。 | 保留为默认工程阈值，但建议通过验证集确定。比如将每个阈值设置为验证集 error 分布的 P75/P90，或按项目容忍误差设置。 |
| `trigger_metrics`                              | 当前只看是否超过阈值，可能导致轻微多项误差叠加时不触发。     | 建议同时使用单项阈值和综合误差：`trigger = any(error_i > threshold_i) OR reference_error_score > score_threshold`。 |
| `heuristic_continue`                           | 逻辑合理，但需要加入“改进不足停止”机制。                     | 建议：`continue = enabled AND round_idx<max_rounds AND candidate_roi_count>0 AND error_reduction>min_gain`。避免无效循环。 |

## 八、决策 Flags 与综合分

| 建议修改指标                | 现有问题                                                     | 科学、可行的修改建议                                         |
| --------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `overall_score`             | 当前 benchmark 模式是越高越好；reference/online 模式可能混入越低越好的 error score，方向不统一。 | 统一成质量分：`benchmark_quality = weighted_mean(AP50, AP75, F1)`；`reference_quality = 1-reference_error_score`；`online_quality = 1-online_risk_score`。最后：`overall_quality_score = weighted_mean(valid_quality_terms)`。 |
| `quality_pass_flag`         | 如果 `overall_score` 未统一到 0–1，则阈值没有意义。          | 先保证 `overall_quality_score ∈ [0,1]`，再设置通过阈值。     |
| `need_param_search_flag`    | 当前只看 overall score 容易误触发。                          | 建议结合错误类型：若碎片化/粘连/边缘伪影高，优先调 stage2 参数；若语义缺口大，优先调 stage1 或主模型。 |
| `need_finetune_flag`        | 当前只看低分容易把参数问题误判为需要微调。                   | 建议加入条件：只有当多轮参数搜索/ROI 细化后仍存在稳定错误，且错误样本质量足够，才进入微调池。 |

| `accepted_improvement_flag` | 如果 overall score 方向不统一，会误判改进。                  | 统一使用 `overall_quality_score` 后再计算：`accepted = new_quality >= old_quality + min_quality_gain`。 |

## 九、微调效果评估类指标

| 建议修改指标                                       | 现有问题                                                     | 科学、可行的修改建议                                         |
| -------------------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `mean_gain_tree_count / crown / closure / density` | `gain = before_error_abs - after_error_abs` 方向正确，但不同指标量纲不同，不能直接混合。 | 分别报告绝对 gain 和归一化 gain：`normalized_gain_i = (before_error_i - after_error_i) / max(before_error_i, eps)`。 |
| `num_improved`                                     | 只统计 `gain>0`，不能体现提升幅度。                          | 保留，同时增加 `mean_normalized_gain`、`median_gain`、`worse_count`。 |
| `stratified_gain`                                  | 合理，但要保证每个地形分组样本数足够。                       | 增加 `n_units`，若某类样本数低于阈值，例如 `<5`，标记为不稳定。 |
| `benchmark_gain.delta`                             | 当前把 AP/Precision/Recall/R² 和 MAE/RMSE 混在一起做 delta，方向容易错。 | 分开定义：高优指标 `delta = after - before`；误差指标 `gain = before - after`。最终综合时统一成“正值代表提升”。 |

# 最建议你采用的统一公式框架

为了让整个表格更科学、可复现，建议所有综合指标都改成下面这套结构：

```text
原始指标 raw_metric
        ↓
可计算性判断 valid / NA
        ↓
方向统一：error 越大越差，quality 越大越好
        ↓
归一化到 0–1
        ↓
缺失项剔除，剩余权重重新归一化
        ↓
输出：
1）标准报告指标
2）业务一致性指标
3）智能体流程决策指标
```

综合评分建议统一为：

```text
reference_error_score = weighted_mean(clamp01(error_i / tolerance_i))

reference_quality_score = 1 - reference_error_score

online_quality_score = 1 - online_risk_score

overall_quality_score = weighted_mean(
    benchmark_quality_score,
    reference_quality_score,
    online_quality_score
)
```

其中 `benchmark_quality_score` 只在有 GT/COCO 标注时使用；`reference_quality_score` 只在有小班参考或人工参考时使用；`online_quality_score` 用于 DOM-only 无标注场景。

## 最终优先修改顺序

| 优先级 | 建议修改内容                                                 |
| ------ | ------------------------------------------------------------ |
| 1      | 统一所有综合分方向：error score 越低越好，quality score 越高越好，不能混用。 |
| 2      | 所有除零风险指标加 `eps` 和 `NA` 逻辑。                      |
| 3      | 删除 `density_error_ratio` 缺失时 `/1000` 的兜底。           |
| 4      | 修正 `duplicate_overlap_ratio` 分母。                        |
| 5      | `closure_error_abs` 前必须统一郁闭度量纲到 0–1。             |
| 6      | 所有固定阈值，如 4m²、6m、CHM>1m，改成可配置或基于参考冠幅/验证集分位数。 |
| 7      | `failure_confidence` 拆成错误严重度和错误类型置信度。        |
| 8      | `benchmark_gain.delta` 区分高优指标和误差指标。              |

一句话总结：**标准精度指标按 COCO/实例分割体系报告；类似小班、调查等矢量数据一致性指标按林业业务变量解释；权重、阈值、综合分全部改成可配置、可校准的工程决策指标，不能写成通用标准公式。**
